# GitHub Issue Resolution

When resolving GitHub issues:

- read the issue closely
- identify the behavioral expectation
- inspect related code before editing
- keep the fix scoped
- add or update tests when behavior changes
- summarize the resolution in operational terms

Do not close the loop with only a code change if the issue implies docs, migration, setup, or verification work.
