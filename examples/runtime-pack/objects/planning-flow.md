# Planning Flow

Before implementation:

- identify the core user loop
- separate product requirements from engineering design
- choose the thinnest useful vertical slice
- define acceptance criteria before coding
- keep future architecture visible without bloating V1

During implementation:

- update the plan as work lands
- verify each slice with tests or runnable proof
- preserve user-owned files and unrelated changes
- document remaining questions explicitly
