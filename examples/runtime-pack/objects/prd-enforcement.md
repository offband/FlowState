# PRD Enforcement

Use the PRD as the source of product truth.

Before adding behavior, check whether it supports the stated product goal.

Reject scope that turns the product into:

- a chat system
- an autonomous agent platform
- a prompt marketplace
- a social product
- a flow execution engine

When implementation pressure creates ambiguity, prefer the path that keeps the system local-first, inspectable, composable, and operationally transparent.
